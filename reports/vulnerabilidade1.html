// 01. VULNERABILITY_OVERVIEW

Foi identificada uma falha critica de Insecure Direct Object Reference (IDOR) na API da plataforma La******ve. O parametro de assinatura X-Sign-Js e nulo/inefetivo, permitindo que atacantes manipulem o token para acessar dados de outros usuarios sem autorizacao.

Target: web.la******ve.com Endpoint: br-uapi.la******ve.com Severity: CRITICAL

Technical Insight:

A Business Logic da API falha ao validar a correlacao entre o token fornecido e a sessao do usuario (_su). O parametro X-Sign-Js deveria garantir integridade, mas aceita requisicoes mesmo quando o token e alterado, permitindo acesso nao autorizado a dados PII de qualquer usuario.

// 03. SECURITY_IMPACT

Confidentiality Breach

Exposicao completa de PII (Personally Identifiable Information) incluindo nome, e-mail, telefone e IDs internos de qualquer usuario corporativo.

Mass Enumeration

Qualquer usuario autenticado pode enumerar a base completa de clientes corporativos atraves da injecao sistematica de tokens validos.

// 04. DEVSECOPS_REMEDIATION

Acao Reativa

Implementar Server-Side Token Validation. O sistema deve vincular o token ao _su (session user) e rejeitar requisicoes onde nao houver correspondencia.

Defesa em Camadas

Ativar validacao criptografica do X-Sign-Js usando HMAC-SHA256. Invalidar requisicoes quando a assinatura nao corresponder aos parametros alterados.
